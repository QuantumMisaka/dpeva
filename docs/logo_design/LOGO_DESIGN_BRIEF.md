# DP-EVA 品牌视觉识别系统设计规范 (Visual Identity System Design Spec)

**版本**: v2.1 (Execution-Ready Brief)  
**日期**: 2026-03-03  
**项目名称**: DP-EVA (Deep Potential EVolution Accelerator)  
**中文名称**: 深度势能演化加速器  
**交付对象**: 高级视觉设计师 / 3D 动效设计师  
**设计目标**: 设计师仅依赖本文件即可落地制作（含VI规范、场景适配与动效/3D方案）

## 1. 品牌核心与设计哲学 (Brand Core)

### 1.1 品牌定位
DP-EVA 是连接 **“严谨物理建模 (DeepModeling)”** 与 **“激进 AI 进化 (Evolution)”** 的桥梁。它在茫茫化学空间（LCL 之海）中，通过主动学习算法，精准筛选出具有独立价值的样本（适格者），加速科学发现的进化过程。

### 1.2 核心隐喻 (Key Metaphors)
*   **秩序 vs 混沌**: 左侧代表科学的秩序与理性，右侧代表进化的爆发与突破。
*   **能量流/加速束 (Acceleration Beam)**: 连接两者的“加速通道”，象征从数据到知识的筛选与放大（避免生物体系联想）。
*   **觉醒 (Awakening)**: 从沉睡的数据中唤醒关键样本，如同 EVA 初号机的觉醒。

### 1.3 项目特征必须可被视觉化 (Project-To-Visual Requirements)
Logo 需要显式或隐式表达以下“项目事实”，以避免仅停留在泛科幻风格：

*   **主动学习闭环**: 从海量样本中筛出少量高价值样本（Precision）。
*   **不确定度与代表性**: 既强调“高不确定度”，也强调“覆盖度/去冗余”（Sampling）。
*   **工程属性**: DP-EVA 是可部署可复现的软件工具，而非仅概念性研究。

### 2.1 CLI Banner 的结构语言（必须被吸收，但不要求ASCII复刻）

```bash

    r" | |_| |  __/_____| |___  \ V /   / ___ \ ",
    r" |____/|_|        |_____|  \_/   /_/   \_" + "\\",

```

具有明确的“终端UI面板”语言：

*   **框体**: 上下边框 + 中间留白 + 核心标题 + 副标题。
*   **居中与秩序感**: 标题主体居中、版面节奏清晰。
*   **配色气质**: 绿色/青色用于强调信息层级（可转换为品牌蓝/荧光绿点亮）。

Logo 在矢量设计中可通过以下方式继承这一语言：

*   **“面板框”可选**: 为主Logo提供一个可切换的“HUD/终端面板框”版本（用于网站Header、启动页、海报），框体为细线或半透明玻璃。
*   **“信息层级”可选**: 在展示级场景加入副标题 `Deep Potential EVolution Accelerator`，在小尺寸应用中仅保留 `DP-EVA`。

## 3. Logo 核心设计规范 (Core Design Elements)

Logo 采用 **“图文融合 (Iconic Logotype)”** 形式，整体为水平结构 `DP-EVA`，并拥有可复用的“图形系统”在不同场景中增强科技感。

### 3.1 结构与栅格系统 (Grid System)
*   **基础网格**: 建立在 `100x100` 单位的模块化网格上。
*   **视觉重心**: 左侧 `DP` 占据 40% 视觉权重，右侧 `EVA` 占据 60%（强调进化）。
*   **字重对比**: `DP` 线条宽度统一（Medium），`EVA` 线条粗细对比强烈（Extra Bold + Hairline）。

补充（用于落地绘制）

*   **基准高度**: 以 `EVA` 的大写高度为 `H`。
*   **线宽**: `DP` 主线宽约 `0.10H`；`EVA` 粗主干约 `0.12H`；`EVA` 极细衬线约 `0.03H`。
*   **圆角/切角策略**: `DP` 采用圆角或六边形切角（60°/120°）；`EVA` 以锐角为主但避免过细断裂。

### 3.2 详细元素定义

#### A. 左侧 `DP` —— 理性的基石
*   **字体**: 基于 **Geometric Sans-Serif** (如 Futura / Rajdhani) 进行定制。
*   **特征**: 
    *   **六边形切割**: 字母 `D` 和 `P` 的转角处采用 60°/120° 切角，致敬苯环与晶体结构。
    *   **闭合性**: 线条封闭，象征严谨的物理守恒律。
*   **色彩**: **DeepModeling Blue**。

#### B. 连接符 `-` —— 加速通道
连接符必须表达“加速/筛选/传递”，建议三选一（默认方案为 1）：

1.  **加速束短划 (Acceleration Beam Dash) — 默认**
    *   形态：细长的短横，中间略亮、两端渐隐（类似粒子加速器束斑）。
    *   细节：短横内部可加入 2–3 条极细平行线（像数据通道），但整体仍像一个 `-`。
    *   寓意：从 `DP`（基础建模）向 `EVA`（进化加速）输送“能量/信息”。

2.  **采样阈门短划 (Selection Gate Dash)**
    *   形态：短横两端略有“门槛”折角（微小的八边形切角），中间留一道缝。
    *   寓意：不确定度阈值 / 过滤边界（AT Field 的理性化抽象），突出“筛选”。

3.  **能量等高线短划 (Energy-Contour Dash)**
    *   形态：由 2 条细线构成的短横，上下线轻微错位（像等势线/能量地形截面）。
    *   寓意：Deep Potential 的“势能地形”被切开后形成的通道，强调“Potential”。

连接符色彩与层级

*   默认用 **Evolution Gradient**（蓝→橙）或 **蓝→紫** 的短渐变，避免绿色主导（绿色仅点睛）。
*   连接符亮度应低于 `EVA` 主体 10–20%，防止喧宾夺主。

#### C. 右侧 `EVA` —— 突破的利刃
*   **字体**: 致敬 EVA 标题的“极粗衬线 + 锐角切割”的气质，但不直接复制原作字形。
*   **特征**:
    *   **E / V**: 拥有极细的横向衬线和极粗的纵向笔画，边缘锐利如刀锋。
    *   **A (核心图形)**: 
        *   **初号机剪影**: 将 `A` 的顶端延伸为初号机的独角 (Horn)。
        *   **AT Field**: 在 `A` 的背景或周围添加微弱的八边形光晕轮廓。
        *   **眼睛**: `A` 的横杠设计为狭长的发光眼部，暗示“筛选与识别”。

版权与致敬边界（必须遵守）

*   `A` 的初号机化应为“抽象剪影/特征提取”，避免复刻原作头部细节（比例、线条、装甲分区）。
*   若使用任何与“EVA标题字体”高度相似的衬线体，必须替换为原创绘制并转曲交付，避免字体版权风险。

## 4. 色彩规范 (Color System)

色彩方案采用 **“冷暖对撞”** 策略，体现理智与激情的冲突美学。

### 4.1 标准色 (Standard Colors)

| 色彩名称 | 角色 | Pantone (PMS) | HEX | RGB | CMYK | 寓意 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **DeepModeling Blue** | 品牌主色 (DP) | 2728 C | `#6E6BFF` | 110, 107, 255 | 70, 60, 0, 0 | 科学、计算、未来 |
| **Warning Orange** | 品牌主色 (EVA) | Orange 021 C | `#FF8A00` | 255, 138, 0 | 0, 60, 100, 0 | 警示、高能、LCL |
| **Neon Green** | 点缀色 (Eyes) | 802 C | `#39FF14` | 57, 255, 20 | 50, 0, 100, 0 | 觉醒、生物能 |
| **Void Black** | 背景色 | Black 6 C | `#0B0B0F` | 11, 11, 15 | 80, 70, 70, 100 | 深空、未知 |

印刷注意事项

*   CMYK 与 Pantone 为“建议值/近似参考”，正式印刷请以设计师打样与纸张/工艺校正为准。
*   `#39FF14` 属高饱和荧光绿，CMYK 直转可能失真，建议在印刷物料中使用 Pantone 荧光专色或降低饱和度的替代色。

### 4.2 辅助色与渐变
*   **Evolution Gradient**: 用于连接符或 `EVA` 字体的填充。
    *   `#6E6BFF` (Blue) → `#FF8A00` (Orange) 的线性渐变，角度 45°。

辅助灰阶（用于UI/线框）

*   **Graphite Gray**: `#2A2A33`（深色线框/面板框）
*   **Mist Gray**: `#C9CAD4`（浅色背景下的线框/分隔线）

## 5. 图形系统（除艺术字外的辅助要素）(Supporting Graphic System)

在不削弱 `DP-EVA` 字样识别度的前提下，允许在字样周围加入“科技前沿感”的背景/徽章元素。原则：**少而精、可裁切、可降级**。

### 5.1 推荐的辅助要素（可组合，但每次最多使用 2 个）

1.  **化学空间点云 (Chemical-Space Point Cloud)**
    *   形态：稀疏点阵/散点云（可带轻微透视）。
    *   语义：海量候选样本。
    *   规则：点云密度从左到右逐渐稀疏；被选中的点用橙色高亮（少量）。

2.  **筛选高亮 (Selection Highlight)**
    *   形态：少量“被圈定的点/簇”，可用小八边形框或细环。
    *   语义：UQ/采样筛选出的高价值样本。

3.  **AT Field 线框 (Octagon Field Frame)**
    *   形态：极细的八边形线框，作为 `EVA` 区域的背板/光晕。
    *   语义：边界、筛选阈值、个体认同。
    *   规则：透明度 10–25%，不得盖住 `EVA` 主体笔画。

4.  **势能等高线 (Potential Contour Lines)**
    *   形态：2–4 条等势线弧线，作为连接符附近的背景纹理。
    *   语义：Deep Potential 的能量地形。

### 5.2 版式与留白（必须遵守）

*   **主视觉优先**: `DP-EVA` 艺术字必须为最高对比度元素。
*   **留白比例**: 主Logo外圈至少保留 `0.5X` 安全间距（见 7.1）。
*   **透明度控制**: 辅助图形默认 10–20% 不透明度；高亮点可达 60–100% 但数量不超过 5 个。
*   **降级策略**: 小尺寸场景（≤24px 高）自动移除所有辅助图形，仅保留 `A` 或 `DP-EVA`。

## 6. 场景适配与变体 (Adaptability)

### 6.1 组合形式
1.  **标准横版**: `DP - EVA` (适用于网站 Header、文档抬头)。
2.  **堆叠竖版**: `DP` 在上，`EVA` 在下，连接符位于中间或作为分割线 (适用于 T 恤、海报)。
3.  **App 图标 (Icon)**: 
    *   **方案 A**: 仅提取机甲化的字母 `A`，背景为深紫/黑色。
    *   **方案 B**: `DP` 字母组成的六边形徽章。

### 6.2 响应式Logo体系（强制交付）

*   **Primary**: `DP-EVA` +（可选）辅助图形（点云/AT Field/等势线）
*   **Secondary**: 仅 `DP-EVA`（无辅助图形）
*   **Mark**: 仅 `A`（初号机化）或 `DP` 几何徽章

### 6.3 单色与反白
*   **墨稿 (Black)**: 用于黑白印刷文档，去除所有渐变，保留线条轮廓。
*   **反白 (White)**: 用于深色图片背景，全白线条，可保留 `A` 的眼睛为透明镂空。

必须提供四套规范样式

*   **标准色**: 具备主色+点睛色+渐变
*   **反白**: 全白（必要时保留少量镂空）
*   **单色**: 单一品牌蓝或单一黑（用于刻蚀/丝印）
*   **黑稿**: 100% K（用于印章/复印）

### 6.4 最小尺寸规范
*   **Web**: 最小高度 24px (确保 `DP` 与 `EVA` 字样清晰)。
*   **Print**: 最小宽度 20mm。
*   **Icon**: 支持 16x16px (Favicon) 识别度（需提供像素对齐的简化版）。

建议导出尺寸（至少覆盖）

*   **App Icon**: `1024`, `512`, `256`, `128`, `64`, `32`, `16`（正方形）
*   **社交头像**: `400x400`, `800x800`
*   **网页Header**: `1200x300`（含面板框可选版）
*   **PPT/文档抬头**: `1920x400`（透明底）
*   **印刷物料**: A4 抬头、名片、贴纸（矢量为主）

## 7. VI 规范与禁忌 (Usage Guidelines)

### 7.1 安全间距 (Clear Space)
*   以字母 `P` 的高度为单位 `X`。
*   Logo 四周需保留至少 `0.5X` 的空白区域，严禁放置其他图文元素。

补充

*   在带“终端面板框”的版本中，面板框与Logo主体之间的内边距至少 `0.35X`。

### 7.2 禁用示例 (Don'ts)
*   🚫 **禁止变形**: 不得对 Logo 进行长宽比拉伸或压缩。
*   🚫 **禁止描边**: 除非在极其复杂的背景上，否则不建议给 Logo 添加外部描边。
*   🚫 **禁止混色**: 不得将 `DP` 改为橙色，或将 `EVA` 改为蓝色（破坏二元对立的寓意）。
*   🚫 **禁止阴影**: 在扁平化场景下，禁止添加投影（3D 渲染场景除外）。

*   🚫 **禁止过度纹理**: 不得使用高噪点/重做旧/赛博故障纹理覆盖文字主体。
*   🚫 **禁止堆叠图形**: 辅助图形一次最多使用 2 个，不得把点云+等势线+AT Field 全部叠加。

## 8. 字体与授权 (Typography & Licensing)

### 8.1 字体策略

*   `DP` 推荐使用 **开源几何无衬线体** 并允许二次改造：`Rajdhani` / `Oxanium` / `Inter` 等（通常为 OFL 授权；最终以设计师核验为准）。
*   `EVA` 推荐 **原创绘制衬线字形**（参考“极粗衬线 + 锐角切割”的气质），避免直接使用带版权风险的商业字体。

### 8.2 交付要求

*   所有字体最终必须“转曲/轮廓化”后随源文件交付。
*   如因项目传播需要保留可编辑文本版本，需提供字体授权说明与购买链接（如适用）。

## 9. 3D 与动态演示方案 (3D & Motion)

### 9.1 3D 材质设定
*   **DP 部分**: **磨砂玻璃 (Frosted Glass)** 或 **阳极氧化铝 (Anodized Aluminum)** 材质，发出微弱的蓝色内发光。
*   **EVA 部分**: **高光黑曜石 (Obsidian)** 外壳，内部包裹着流动的 **岩浆/LCL 液体** (橙色自发光)。

3D 预览渲染规范（用于一致性）

*   视角：等距或轻微透视（FOV 25–35）。
*   光照：主光冷色、辅光暖色，形成冷暖对撞；背景保持 `#0B0B0F`。
*   细节：`A` 的“眼睛”可发出荧光绿，亮度不超过主体高光。

### 9.2 动态演示脚本 (Motion Concept)
1.  **Start**: 屏幕全黑，出现绿色的 ASCII 字符流（致敬项目 Banner 代码），快速滚动。
2.  **Glitch**: 字符流出现故障 (Glitch) 效果，画面撕裂。
3.  **Evolve**: 
    *   左侧汇聚出蓝色的几何线条，构建出 `DP`。
    *   右侧瞬间爆发橙色光芒，`EVA` 字母如利刃般切开黑暗，`A` 的眼睛亮起。
4.  **Loop**: 连接符 `-` 变为短促的“加速束脉冲”，光流在左右两端往复传输；背景点云中的少数点被高亮“选中”。
