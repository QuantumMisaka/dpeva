# v0.8.0 ATST 集成准备计划

- Status: active
- Audience: Developers / Maintainers
- Last-Updated: 2026-06-10
- Source Draft: `.trae/documents/2026-06-10-dpeva-atst-integration-implementation-plan.md`

本文档是 v0.8.0 开发周期的项目级入口，用于承接项目管理者给出的 ATST 集成实施计划。当前阶段只做实现前准备，不实现内部 writer、exploration 抽象层或 `ATSTToolsBackend`。

## 1. v0.8.0 范围

v0.8.0 后续开发围绕两个方向展开：

1. 移除 DPEVA Labeling 主链路对 GitLab `ase-abacus` fork 的运行时绑定。
2. 为 `atst-tools` 建立可选 exploration backend 边界，支持未来结构探索结果进入 DPEVA 的 `feature -> UQ -> DIRECT -> label` 主链路。

## 2. 已并入 v0.8.0 的 v0.7.2 后变更

- 模型对比报告生成器：`scripts/build_model_comparison_report.py`
- DPA4 测试环境变量模板：`scripts/env/dpeva-dpa4-test.env`
- DeepMD 开发版本号解析兼容修复：`src/dpeva/utils/env_check.py`
- 模型对比报告生成器单测：`tests/unit/test_model_comparison_report.py`
- 文档治理自动化加固：`scripts/doc_check.py`、`scripts/check_docs_freshness.py`、`scripts/verify_docs.sh`
- docs/reports 去冗余与生命周期治理：`docs/reports/README.md` 与相关 reports/archive 索引

## 3. 后续实现顺序

后续实现应按源计划中的 PR 拆分推进：

1. PR-1：Labeling 去 `ase-abacus`
   - 新增 DPEVA 内部最小 ABACUS writer。
   - 替换 `generator.py` 中对 `ase.io.abacus` 的导入和调用。
   - 补齐 writer 单测与 golden output 回归。
2. PR-2：exploration 抽象层
   - 新增 `src/dpeva/exploration/base.py` 与 `manager.py`。
   - 明确 exploration 与 sampling 的职责边界。
3. PR-3：`ATSTToolsBackend`
   - 以可选依赖方式接入 `atst-tools`。
   - 首版只覆盖 `md` / `relax`。
   - 通过 CLI 或稳定公开接口调用，不深度耦合 `atst-tools` 内部模块。

## 4. 非目标

当前准备阶段不做以下事项：

- 不新增 `src/dpeva/labeling/abacus_io.py`
- 不修改 `src/dpeva/labeling/generator.py` 的运行时逻辑
- 不新增 `src/dpeva/exploration/`
- 不向主依赖加入 `atst-tools`
- 不把 `ase-abacus` 并入项目依赖

## 5. 验收要求

准备阶段完成后应满足：

- 项目版本标识已进入 `0.8.0`
- v0.7.2 之后的已并入变更在开发手册 v0.8.0 条目中有记录
- ATST 集成计划在 `docs/plans/` 下有项目级入口
- 文档治理门禁可继续通过
