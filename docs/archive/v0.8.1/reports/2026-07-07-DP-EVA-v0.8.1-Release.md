# DP-EVA v0.8.1 Release Notes

- Status: archived
- Audience: Users / Developers / Maintainers
- Last-Updated: 2026-07-07
- Version: 0.8.1
- Base branch: `main`
- Release tag: `v0.8.1`

DP-EVA v0.8.1 closes the SAI Slurm-native labeling cycle. This release focuses on large FP11 first-principles labeling reliability, Slurm array submission, task-class launcher correctness, and the documentation/archive records required to make the operational evidence reproducible.

## Highlights

### Slurm-Native Labeling Arrays

Labeling can now submit homogeneous task groups as Slurm arrays instead of one `sbatch` per packed bundle. The implementation adds:

- `submission.slurm_array` and `submission.slurm_array_task_limit` configuration fields.
- Manifest-backed array tasks through `dpeva.submission.array`.
- Shared Slurm job name normalization through `dpeva.submission.names`.
- `JobManager.submit_array()` and `JobConfig.array` rendering.
- Labeling workflow grouping by task class and attempt.

This keeps large active-learning labeling runs closer to the scheduler model and avoids thousands of independent submission calls.

### SAI / Slurm Monitoring Safety

The labeling monitor now treats an empty `squeue` result as insufficient evidence for completion and checks `sacct -X` active states before concluding that Slurm jobs are finished. This prevents transient queue-query gaps from triggering premature retries and duplicate writes on long-running array jobs.

### Task-Class Launcher Boundary

v0.8.1 records and enforces the launcher responsibility boundary used on SAI:

- Ordinary `launcher_mode="abacus"` tasks do not read `MAP_OPT`.
- Only explicit `launcher_mode="mpi_abacus"` task classes source the SAI rank-map script and use `--map-by $MAP_OPT`.
- SAI profile/module setup ordering is documented in the Slurm and configuration guides.

### FP11 SAI-1344 Production Record

The FP11 SAI-1344 execution record is archived under v0.8.1. The final record covers recovery, extract/postprocess, stats generation, and backend reporting. The supporting scripts under `scripts/fp11_1344_*` provide config generation, backend report generation, recovery after false finish, and finalization helpers for this operational path.

### Documentation Governance

The release archive includes:

- docs audit / DP-EVA operator skill plan.
- dataset audit plan and design spec snapshot.
- Slurm array backend implementation plan snapshot.
- FP11 SAI-1344 production execution record.
- this v0.8.1 release note.

## Upgrade Notes

- Existing non-array labeling behavior remains the default. Enable Slurm arrays with `submission.slurm_array=true`.
- Use `submission.slurm_array_task_limit` to render a Slurm array throttle such as `--array=0-4667%128`.
- For SAI ABACUS MPI jobs, use task classes with `launcher_mode="mpi_abacus"` and the documented SAI environment setup ordering.

## Validation

Release validation used the `dpeva-dpa4` environment and the project documentation gates:

- `bash -n scripts/fp11_1344_finalize_after_execute.sh scripts/fp11_1344_recover_wrapper.sh`

- `pytest` for Slurm submission, labeling workflow, and FP11 helper script tests
- `python scripts/doc_check.py`
- `python scripts/check_docs_freshness.py --days 90`
- `make -C docs html SPHINXOPTS="-W --keep-going"`
