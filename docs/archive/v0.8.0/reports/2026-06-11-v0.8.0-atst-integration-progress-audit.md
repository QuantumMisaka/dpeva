# 2026-06-11 v0.8.0 ATST 集成计划进度审查

- Status: archived

本文档用于开发人员对照此前 v0.8.0 ATST integration 计划，确认当前完成进度、剩余缺口与后续动作优先级。结论基于当前仓库中的实际代码、测试、配置、CI 与文档证据，而不是仅依据计划文本自述。

> 2026-06-11 follow-up：本报告发布后的收尾补强已补齐 golden baseline、exploration manifest、README/installation/examples 入口、CI explore smoke 与 v0.8.0 最小集成验收。`build/lib` 残留经复核为 `.gitignore` 忽略的本地构建产物，不属于 Git 跟踪源码；当前工作区已清理 `build/`。

## 1. 综合结论

- 当前状态适合定义为：**核心功能已落地，v0.8.0 最小验收闭环已完成**。
- Labeling 主链路去 `ase-abacus` 的核心代码改造已基本完成，内部最小 ABACUS writer 已接管主实现。
- exploration 抽象层、`ATSTToolsBackend` 首版与 `dpeva explore` CLI 已落地，首版 `md/relax` 路径具备最小可运行骨架。
- v0.8.0 后续不再建议横向扩展 scope；`feature -> UQ -> DIRECT -> label` 的正式自动接线保留为 0.8.x 后续架构任务。
- 本地 ignored `build/` 构建产物已清理，避免人工审查误判。

## 2. 验收口径

- **已完成**：代码、配置、测试或文档证据已形成闭环，可作为当前阶段验收通过。
- **部分完成**：核心实现已存在，但仍有明确缺口、残留、未验证项或收尾动作未完成。
- **未完成**：计划要求的关键动作或验收证据尚未形成。

3. 在后续版本中决定 exploration 与 `feature -> UQ -> DIRECT -> label` 的正式接线策略。
