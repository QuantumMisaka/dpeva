以文章为主线：DPOSE 到底要解决什么

论文提出的四个 UQ 要求

文章明确说，好的 UQ 框架应当同时满足四点：误差估计和真实误差相关、计算开销低、易于加到任意 regression 模型上、能直接而稳健地传播到派生量。这个定位见 DPOSE 论文 full.md:63-73。

DPOSE 的关键不是“又一个 ensemble”，而是找到一个可用于原子模拟实际后处理的 ensemble 表达，使误差能穿过总能、平均量、热物性和主动学习筛选。

为什么不能只给一个 σ

原子模拟经常不是直接消费模型输出，而是消费输出的组合：结构总能、force、轨迹平均、空位形成能、热容等。文章指出这些任务需要能组合 uncertainty 的方法，尤其在 MLIP 轨迹中，误差估计必须能对平均值和派生量传播，见 full.md:69-73、full.md:420-422。

DPOSE 的核心折中

它使用共享 backbone、只扰动/训练最后层权重的 shallow ensemble。相比深 ensemble，训练和推理更便宜；相比单点 variance 预测，它能保留成员相关性并直接传播成员预测。

主动学习语境

论文用 water surface 和 QM9 inconsistent structures 展示 OOD 检测：DPOSE 的 uncertainty 能区分高误差结构，并通过加入少量结构改善预测和校准，见 full.md:395-414。

DPOSE / LLPR 具体运作逻辑

1. 训练一个 base 模型 保留主体网络表示。对 DP-EVA 来说，这个 base 模型是 DeepMD/DPA 模型。
2. 抽取 last-layer feature 对每个结构得到最后线性层之前的特征 f(A)。这是 LLPR 的统计对象，不应混同于普通 descriptor。
3. 构建后验尺度 累积训练集 F^T F，加入正则 λI，用 Cholesky 或 inverse 表达最后层权重的不确定性。
4. 校准全局倍率 用 validation residual 对 uncertainty 乘一个 α。metatrain 支持 squared residuals、absolute residuals、CRPS。
5. 生成/使用 ensemble 围绕真实最后层权重采样 shallow ensemble，预测后重居中到 base 模型输出，再用成员 spread 做传播。

数学对象的最小闭环

训练特征: F = [f(A_1), ..., f(A_N)]^T
LLPR covariance: C = F^T F + λ I
解析 uncertainty: σ(A*) = α sqrt(f(A*)^T C^{-1} f(A*))
ensemble weights: w_k = w_base + ε_k,  ε_k ~ N(0, α² C^{-1})
raw ensemble: y_k(A*) = f(A*) · w_k
re-centered ensemble: y_k' = y_k - mean_k(y_k) + y_base(A*)
DP-EVA 适配点：DeepMD 的公开路径给出的是 fitting last-layer feature；force DPOSE 需要 ensemble energy 对坐标保持可微并求梯度。当前 DP-EVA energy 路径是严格可用的，force 路径不能从 detached feature 推导，因此保持 fail-fast。

DP-EVA 当前实现怎么落地

核心数值层：src/dpeva/uncertain/llpr.py

LLPRState.from_training_features 聚合 last-layer feature，构建 F^T F + λI 并做 Cholesky（llpr.py:36-60）。predict_uncertainty 用 Cholesky solve 计算 sqrt(f^T C^-1 f) 并乘 α（llpr.py:62-74）。状态可保存/加载（llpr.py:86-117）。

校准实现覆盖 squared residuals、absolute residuals、CRPS，并新增 NLL/CRPS/coverage benchmark（llpr.py:120-166、llpr.py:233-304）。

ensemble 层：src/dpeva/uncertain/dpose.py

DPOSEEnsemble.sampled_weights 校验真实权重维度，并从 α² C^-1 采样最后层权重（dpose.py:33-49）。sample_energy_ensemble 计算成员能量并重居中到 base energy（dpose.py:51-62）。

resolve_last_layer_weights 支持显式 .npy 权重，也能从 PyTorch checkpoint/state_dict 按 feature dimension 和 head 匹配 2D weight；无匹配时列出候选 shape（dpose.py:142-217）。

workflow 层：collect.py

当 uq_backend="llpr" 时，collect 加载训练和候选 last-layer feature，按 dataname 对齐候选池（collect.py:214-249）。请求 ensemble 时加载 base energy、解析权重、设置默认 energy_ensemble.npy 路径（collect.py:250-284）。

输出 CSV 包含 analytic LLPR 和 DPOSE summary columns，并允许按 energy_ensemble_std_per_atom 采样（collect.py:292-328、collect.py:405-417）。

配置层：src/dpeva/config.py

新增字段覆盖模型路径、head、权重路径、base energy、状态复用、ensemble 输出和 collect score（config.py:470-497）。strict parity 校验要求 ensemble 请求必须提供 base energy 与真实权重来源（config.py:528-548）。

与文章和 Atomistic Cookbook 应用场景的对应关系

主动学习 / 采样

文章强调 DPOSE 可识别 OOD 高误差结构并驱动 active learning（full.md:407-414）。DP-EVA 的 collect workflow 已能按 energy_uncertainty_per_atom 或 energy_ensemble_std_per_atom 过滤候选，结合现有 DIRECT/2-DIRECT 采样。

适配程度：高。当前 DP-EVA 最自然的落点就是主动学习候选池筛选。

训练一个带 UQ 的势

cookbook llpr-from-scratch 明确展示“训练 baseline potential 并装备 LLPR 与 shallow last-layer ensemble”（llpr-from-scratch.md:10-20）。DP-EVA 的实现对应后半段：在 DeepMD base 模型和 last-layer feature 已存在时，构建 LLPR state 与 ensemble。

适配程度：中高。DP-EVA 侧还需把 feature 生成、base energy、权重导出打通成更完整 recipe。

空位形成能 / 派生量传播

PET-MAD-UQ recipe 包含 vacancy formation energy uncertainty（pet-mad-uq.md:19-24）。DP-EVA 输出的 energy_ensemble.npy 可以离线传播到 E_vac - E_bulk + μ 这类组合量，只要所有结构行顺序和 ensemble member 对齐。

适配程度：中。需要一个小的后处理工具把多个结构集合的 ensemble 做组合。

MD 轨迹平均与热物性

文章和 PET-MAD-UQ 都强调对 MD 平均量、reweighting、i-PI 的不确定度传播（full.md:385-391、pet-mad-uq.md:21-28）。DP-EVA 目前没有在线 calculator，但 ensemble 文件能支撑离线轨迹帧级统计。

适配程度：中低。下一步应提供 trajectory-level aggregation/reweighting 脚本。

样本选择与结构多样性

cookbook sample-selection 使用 FPS/CUR 在 descriptor 空间选样（sample-selection.md:10-25）。DP-EVA 可以把结构多样性采样和 DPOSE uncertainty 结合：先用 UQ 选高风险区间，再用 DIRECT 保持结构覆盖。

适配程度：高。DP-EVA 原本就有 descriptor sampling，LLPR/DPOSE 是任务相关 UQ 轴。

OOD 审计

文章的 water surface、QM9 inconsistent structures 都是 OOD 审计示例。DP-EVA 的 CSV summary 可以对每个候选 frame 输出 uncertainty range、ensemble std、identity 分类，适合写审计报告。

适配程度：高。需要真实标注后用 calibration_benchmark 评估 residual/uncertainty 对齐。

边界、风险和建议讨论点

1. Feature 语义必须保持 last-layer

如果用户把普通 descriptor 当作 llpr_train_feature_dir 输入，代码仍能计算矩阵，但语义会退化为 descriptor-space density proxy。建议在 feature generator 的输出 manifest 中写入 feature_kind=fitting_last_layer 并在 collect 读取时校验。

2. Weight resolver 依赖 checkpoint 内部结构

显式 llpr_last_layer_weights_path 最稳。自动从 checkpoint 按 shape/head 匹配是工程便利，但 DeepMD PyTorch checkpoint 字段不是稳定公开 API。建议 GPU 集成测试覆盖每个目标 DPA3/DPA4 fixture。

3. Energy ensemble 不等于 force ensemble

当前 energy_ensemble.npy 是 frame-level energy ensemble。force-level DPOSE 必须通过可微 PyTorch graph 对每个 ensemble energy 求坐标梯度。detached feature 路径不能推导 force uncertainty。

4. 校准需要真实 residual

LLPR raw uncertainty 可用于排序；要把它解释成可信误差条，需要 labeled validation residual。DP-EVA 已有 LLPRCalibrator 和 benchmark，但 collect 的自动校准数据接入仍应单独补。

5. 系统大小归一化要随目标语义确认

metatrain 对 system target 用原子数归一化 features；DP-EVA 支持 llpr_feature_normalization="mean" 并默认使用。不同 DeepMD head/输出若语义不同，应在真实 benchmark 中确认。

6. Proper scoring ensemble training 仍缺

文章和 metatrain 支持 NLL/CRPS 训练 ensemble 权重。DP-EVA 当前是 analytic LLPR-sampled ensemble；对需要更强校准的生产应用，应规划 frozen-backbone shallow trainer。
