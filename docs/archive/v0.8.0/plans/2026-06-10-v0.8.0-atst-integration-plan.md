# v0.8.0 ATST 集成计划

- Status: archived
- Audience: Developers / Maintainers
- Last-Updated: 2026-06-11

本文档是 v0.8.0 开发周期的项目级入口归档，用于记录 ATST integration 计划、验收边界与闭环状态。当前已完成 Labeling 去 `ase-abacus`、内部 ABACUS writer、exploration 抽象层与首个 `ATSTToolsBackend` 的最小实现。

## 1. v0.8.0 范围

v0.8.0 后续开发围绕两个方向展开：

1. 移除 DPEVA Labeling 主链路对 GitLab `ase-abacus` fork 的运行时绑定。
2. 为 `atst-tools` 建立可选 exploration backend 边界，支持未来结构探索结果进入 DPEVA 的 `feature -> UQ -> DIRECT -> label` 主链路。

## 2. 已并入 v0.8.0 的 v0.7.2 后变更

- 模型对比报告生成器：`scripts/build_model_comparison_report.py`
- DPA4 测试环境变量模板：`scripts/env/dpeva-dpa4.env`
- DeepMD 开发版本号解析兼容修复：`src/dpeva/utils/env_check.py`
- 模型对比报告生成器单测：`tests/unit/test_model_comparison_report.py`
- 文档治理自动化加固：`scripts/doc_check.py`、`scripts/check_docs_freshness.py`、`scripts/verify_docs.sh`
- docs/reports 去冗余与生命周期治理：`docs/reports/README.md` 与相关 reports/archive 索引

## 3. 已落实内容

1. Labeling 去 `ase-abacus`
   - 新增 `src/dpeva/labeling/abacus_io.py`，提供 DPEVA 内部最小 ABACUS INPUT/KPT/STRU writer。
   - 替换 `generator.py` 中对 `ase.io.abacus` 的导入和调用。
   - 补齐 writer、generator 与 import 回归单测。
2. Exploration 抽象层
   - 新增 `src/dpeva/exploration/base.py`、`manager.py` 与结构读取辅助。
   - 明确 exploration 与 sampling 的职责边界，sampling 不直接耦合 ATST。
3. `ATSTToolsBackend`
   - 以 `dpeva[explore]` 可选依赖方式接入 `atst-tools>=2.1.0`。
   - 首版只覆盖 `md` / `relax`。
   - 新增 `dpeva explore` CLI，通过 `atst run CONFIG` 调用后端。

## 4. 收尾验收补强

2026-06-11 进度审查后，v0.8.0 收尾补强聚焦可验收交付，不扩展 workflow scope：

1. Labeling golden baseline
   - 固化 `INPUT/KPT/STRU` golden 输出。
   - 新增文本级契约测试，防止 ABACUS writer 输出格式漂移。
2. Exploration 结果交付
   - `ATSTToolsBackend` 消费 `input_structures`，写入 `work_dir/dpeva_inputs/` 作为 DPEVA 侧输入快照。
   - 每次运行写入 `work_dir/dpeva_exploration_result.json` manifest。
   - 对配置的 `result_structure_paths` 做存在性与 ASE 读取校验，失败时返回标准 `failed` result。
3. 入口与门禁
   - README、installation guide 与 examples recipes 公开 `dpeva[explore]` 安装和 `dpeva explore` 用法。
   - CI 增加 `.[explore]` 安装与 CLI smoke check。
   - `build/` 属于 ignored 本地构建产物，发布前必须清理，但不作为 `src/` 主链路替换失败证据。

## 5. 非目标

当前 v0.8.0 最小实现仍不做以下事项：

- 不向主依赖加入 `atst-tools`
- 不把 `ase-abacus` 并入项目依赖
- 不实现 NEB/AutoNEB/Dimer/IRC 等 ATST workflow
- 不把完整 `abacuslite` 并入 DPEVA
- 不在 v0.8.0 中完成 exploration 到 `feature -> UQ -> DIRECT -> label` 的自动主链路接线

## 6. 验收要求

完成后应满足：

- 项目版本标识已进入 `0.8.0`
- 标准 ASE 环境下 `import dpeva.labeling` 不依赖 `ase.io.abacus`
- `ase>=3.28.0` 为核心依赖，`atst-tools` 仅在 `explore` extra 中
- `dpeva explore` 可通过 `ATSTToolsBackend` 调用 `atst run`
- Labeling golden baseline 测试覆盖 `INPUT/KPT/STRU`
- `dpeva explore` backend 产生 manifest、输入结构快照和标准失败结果
- README、installation、examples 与 CI smoke 覆盖 `dpeva[explore]`
- 文档治理门禁可继续通过
