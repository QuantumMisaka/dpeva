## 1. 背景与目标

基于 v0.6.0 开发阶段的审查，发现文档集存在严重的结构滞后与内容缺失问题。尽管 `docs/policy/docs-structure.md` 定义了良好的规范，但实际执行存在偏差。本计划旨在在 v0.6.0 发布前彻底解决这些治理债，确保文档集的一致性、完整性与权威性。

**核心目标**：
1.  **单一权威来源 (SSOT)**：确保 `config_schema.md` 完整覆盖所有配置项，消除代码与文档的脱节。
2.  **边界清晰**：严格执行 Active/Archive 分离，清理干扰视线的历史文件。
3.  **索引准确**：更新结构树与元信息，确保开发者能快速定位最新规范。

## 2. 现状问题分析 (Gap Analysis)

| 问题类别 | 具体表现 | 严重程度 |
| :--- | :--- | :--- |
| **内容缺失** | `docs/reference/config_schema.md` 仅包含 Labeling 配置，缺失 Train/Infer/Collect/Analysis/Feature 等核心模块配置。 | **Critical** |
| **内容缺失** | `.trae/skills/analysis.md` 缺失，导致 Agent 无法正确引导用户使用 Analysis 新功能。 | **High** |
| **归档不净** | `docs/reports/` 下存在大量 2026-02 的历史报告；`docs/plans/` 包含旧版 `visualization_decoupling_plan.md`。 | **Medium** |
| **索引滞后** | `docs/policy/docs-structure.md` 中的文件树未包含 v0.6 新增的计划文档与报告。 | **Medium** |
| **元信息缺失** | 部分 `active` 文档（如 `config_schema.md` 自身）缺少 `Status/Audience/Last-Updated` 头。 | **Low** |

## 3. 执行任务清单 (Action Items)

### 3.1 权威参考文档补齐 (Reference)

- [x] **重构 Config Schema**：基于 `src/dpeva/config.py`，全量重写 `docs/reference/config_schema.md`。
    - 需包含：`TrainingConfig`, `InferenceConfig`, `CollectionConfig`, `LabelingConfig` (更新 integration 字段), `AnalysisConfig` (新增 mode 字段), `FeatureConfig`。
    - 格式：统一使用表格形式，标注类型、默认值、必填项与说明。
- [x] **创建 Analysis Skill**：新建 `.trae/skills/analysis.md`。
    - 内容：包含 `dataset` 模式与 `model_test` 模式的配置示例、CLI 调用方式与常见错误排查。

### 3.2 归档与目录清理 (Archive & Cleanup)

- [x] **归档历史报告**：

    - 将 `docs/reports/modulo-hypothesis.md` 移动至 `docs/archive/reports/`（若确认为旧假设）。
- [x] **归档旧计划**：

- [x] **治理 Governance 目录**：
    - 将 `docs/governance/audits/2026-02-*.md` 移动至 `docs/archive/governance/audits/`。

### 3.3 索引与元信息更新 (Index & Meta)

- [x] **更新结构树**：运行 `tree docs` 并更新 `docs/policy/docs-structure.md` 中的文件列表。
- [x] **补齐元信息**：检查所有 `docs/guides/` 和 `docs/reference/` 下的文件，确保包含标准 YAML 头。
- [x] **更新 README**：在 `docs/README.md` 中更新指向 `config_schema.md` 和新 Skill 的链接。

## 4. 验收标准 (Acceptance Criteria)

1.  **Config 覆盖率 100%**：`config_schema.md` 包含所有 Pydantic Config 类的字段定义，且与代码一致（特别是 `LabelingConfig.integration_enabled` 和 `AnalysisConfig.mode`）。
2.  **根目录整洁**：`docs/reports` 和 `docs/plans` 下仅包含 v0.6 相关的 Active 文件。
3.  **Skill 可用**：`analysis` skill 文件存在且内容正确。
4.  **结构树一致**：`docs-structure.md` 与实际目录结构完全对应。

## 5. 风险控制

- **文档与代码再次脱节**：在 CI 中增加文档检查步骤（未来规划），当前通过人工 Review 保证。
- **归档误删**：仅移动文件，严禁删除任何历史文件。
