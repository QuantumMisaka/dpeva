## 2. 配置契约：Analysis 双模式

### 2.1 模式定义

- `model_test` 模式：沿用当前 `result_dir` 输入，解析 `dp test` 结果。  
- `dataset` 模式：新增数据集输入字段，直接统计 deepmd 数据。  

### 2.2 字段契约

| 字段 | 类型 | 适用模式 | 兼容策略 |
| :--- | :--- | :--- | :--- |
| `result_dir` | Path | model_test | 保持必填，不变 |
| `output_dir` | Path | all | 保持默认值逻辑 |
| `type_map` | List[str] | all | 保持必填 |
| `data_path` | Optional[Path] | model_test | 保持可选 |
| `ref_energies` | Dict[str, float] | all | 保持默认空 dict |
| `mode` | Literal[`model_test`, `dataset`] | all | 新增；默认 `model_test` |
| `dataset_dir` | Optional[Path] | dataset | 新增；mode=dataset 时必填 |

### 2.3 约束

1. `mode=model_test` 时，`result_dir` 必须存在；忽略 `dataset_dir`。  
2. `mode=dataset` 时，`dataset_dir` 必须存在；允许 `result_dir` 空置。  
3. 旧配置（无 `mode`）按 `model_test` 处理，行为不变。

## 3. 配置契约：Labeling Integration MVP

### 3.1 字段契约（新增）

| 字段 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| `integration_enabled` | bool | false | 是否在 labeling 末端启用自动整合 |
| `existing_training_data_path` | Optional[Path] | None | 旧训练集路径，空表示仅导出新标注数据 |
| `merged_training_data_path` | Optional[Path] | None | 合并输出路径，空时走默认规则 |
| `integration_deduplicate` | bool | false | 是否启用去重（MVP 默认关闭） |

### 3.2 约束

1. `integration_enabled=false` 时，工作流行为与 v0.5.3 完全一致。  
2. `integration_enabled=true` 且 `existing_training_data_path` 为空时，仅导出新标注结果，不报错。  
3. `integration_enabled=true` 且提供旧训练集时，必须输出可供 train 直接使用的数据目录。

## 5. 文档边界策略（active/archive/spec）

### 5.1 active 与 archive

1. `docs/archive/*` 仅保留历史上下文，不作为现行规范来源。  
2. 现行规范仅在 `docs/policy/*`、`docs/reference/*`、`docs/guides/*` 维护。  
3. 同主题若 active 与 archive 并存，active 文档必须显式指向唯一权威入口。  

### 5.2 spec 落点策略

1. v0.6 不做目录迁移。  
2. 现行执行型规格统一落在 `docs/plans/*`，通过状态字段区分 active/closed。  
3. 历史规格继续保留在 `docs/archive/specs/*`，不回流到现行入口。
