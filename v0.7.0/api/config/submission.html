Common & Submission Configuration#

pydantic model dpeva.config.BaseWorkflowConfig[源代码]#

Base configuration for all workflows.

field dp_backend: str = 'pt'#

DeepMD-kit backend (e.g. 'pt', 'tf', 'jax', 'pd').

field omp_threads: int | Literal['auto'] = 4#

OpenMP threads (set to 'auto' to use all cores).

field submission: SubmissionConfig [Optional]#

Submission configuration.

field work_dir: Path [Optional]#

Working directory.

validator extract_flat_submission_config  »  all fields[源代码]#

Allow flat config to populate nested submission config for backward compatibility. e.g. {'backend': 'slurm'} -> {'submission': {'backend': 'slurm'}}

validator validate_dp_backend  »  dp_backend[源代码]#

Validates that the DeepMD backend is supported.

validator validate_omp_threads  »  omp_threads[源代码]#

Validates OMP thread count and resolves 'auto'.

pydantic model dpeva.config.SubmissionConfig[源代码]#

Configuration for job submission.

field backend: Literal['local', 'slurm'] = 'local'#

Execution backend.

field env_setup: str | List[str] = ''#

List of shell commands to execute before running the task (e.g., module load cuda).

field slurm_config: Dict[str, Any] [Optional]#

Slurm specific parameters (partition, nodes, etc).

validator validate_env_setup  »  env_setup[源代码]#

Converts list of commands to a newline-separated string.
